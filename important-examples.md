# 重要性判断规则

> **目标读者:Claude 本身**。在 daily-fetch 时给每条新闻打 ⭐ 评级,在 weekly-report 时决定哪些条目入选最终周报。
> 这个文件会**持续追加**——每次 Long 划完重点后,说"学习这次的判断",Claude 把新规律追加到本文件末尾的"学习记录"区块。

---

## 评级定义

| 评级 | 含义 | 行动 |
|------|------|------|
| ⭐⭐⭐ | 高重要,必入周报 | 默认放入周报,且通常排在前面 |
| ⭐⭐ | 中等重要,建议入周报 | 由 Long 决定;如果 ⭐⭐⭐ 条目较多,可能被挤掉 |
| ⭐ | 低重要,备选 | 默认不入周报,仅作记录 |
| (无星) | 不重要 | 仅在 daily 文件保留,不进入 pool |

## 默认加权规则(公司维度)

### 头部公司默认 +1 级
以下公司的任何实质性动作,**默认评级 +1**(比如本来是 ⭐⭐ 的事件,遇到这些公司自动升为 ⭐⭐⭐):
- **Waymo**(行业风向标,开城、融资、事故、监管都关键)
- **Tesla**(FSD 版本、Cybercab、欧洲/中国落地、Musk 公开表态)
- **小马智行**、**文远知行**、**萝卜快跑**(中国 L4 头部)

### 其他重点公司默认评级
Zoox、Uber、Wayve、小鹏、华为鸿蒙、Momenta、元戎、地平线:默认按事件本身判断,不自动加权。

### 其他公司
默认 ⭐,除非事件本身非常重大(融资超过 5 亿美元、区域监管牌照、重大事故等)才升级。

## 默认加权规则(事件类型维度)

### ⭐⭐⭐ 默认级别(几乎必入周报)
- **区域扩张/开城**:新增商业化运营城市、新增国家落地、运营区域扩大
  - 例:Waymo Nashville 开城、小马 Zagreb 商业化
- **重大商业化里程碑**:首次收费、首个 xxx、首单、盈亏平衡
  - 例:小马 Gen-7 单车 UE 盈亏平衡
- **重大融资 / 上市**:超过 5 亿美元融资、IPO 递表、IPO 定价
- **重大事故 / 安全事件**:涉及人员伤亡、大规模车辆停运、监管调查启动
  - 例:武汉萝卜快跑集体停运事件、NTSB 对 Waymo 校车问题的调查
- **新产品 / 新车型发布**:Robotaxi 新车型、Cybercab 量产、下一代系统
- **重大技术发布**:世界模型、新一代大模型、端到端架构升级(只要是头部公司)
  - 例:PonyWorld 2.0、Waymo World Model、小鹏 VLA 2.0

### ⭐⭐ 默认级别(看情况)
- **FSD 大版本推送**(v14.3 这种),小版本(v14.3.1)不算
- **战略合作签约**:跨国合作、OEM 战略绑定
- **监管动向**(已通过的法规),**仅提案**的降一级
- **高管任命/离职**:仅限头部公司 CTO/CEO/智驾负责人
- **产能/车队规模数据**:季度/年度官方数据披露

### ⭐ 默认级别(通常不入周报)
- 路测牌照新获得(除非是新地区首张)
- 小版本软件更新
- 社区/爱好者目击事件(非事故)
- 合作意向 / MOU 签署(非正式合同)

### 不评级(仅记录)
- 公司营销活动、广告、赞助
- 纯股价/财报分析(除非财报里有新战略)
- 旧事件复述、盘点回顾文章

## 合并与去重规则(同公司多条事件)

如果一家公司本周有 2+ 条新闻:
1. **先按事件类型分组**(区域扩张归一组、技术发布归一组、融资归一组)
2. **同组内的事件合并成一个主条目**,用分号串联 headline,子条目展开
3. **组间分开**,一家公司可以有多个主条目(但尽量控制在 1-2 个)
4. **整体重要性取最高**的那一条

## 被淘汰的常见陷阱(易错情况)

### 跨界合作 / 非核心业务
- Waymo + Waze 坑洼数据(跨界,非核心运营)→ 仍算 ⭐⭐,但在周报里排后
- Waymo + DoorDash 外卖配送试点 → ⭐
- Tesla 主题歌发布 / Cybercab 涂装更新 → 不评级

### "可能"/"计划"类模糊新闻
- "据知情人士透露 X 将在 Y 时间进入 Z 国"→ ⭐,除非有具体证据
- "X 计划在未来 5 年内达到 Y 规模"→ 不评级,除非是公司 CEO 正式发言

### 负面技术细节 / hack 新闻
- "黑客通过 X 方法激活 FSD"→ ⭐
- "某用户吐槽 FSD 在 X 场景表现差"→ 不评级

### 行业分析文章 / 盘点
- "2026 年智驾市场竞争格局分析"→ 不评级(这是文章,不是新闻)
- "X 家公司累计亏损 Y 元"→ 不评级,除非伴随重大战略调整

## 学习记录(持续追加)

> 格式:每次 Long 给出反馈后,在这里记一条,包括日期、反馈内容、规则总结。

### 2026-04-10(初始版,基于测试反馈)
- **Long 反馈**:"Waymo 和 Tesla 的任何动作都比较重要,尤其是区域扩大这种新闻,比如 Waymo 去 Nashville 的邀请制"
- **规则提炼**:Waymo 和 Tesla 的区域扩张默认 +1 级;Nashville 这种开城类新闻就算不是新大区也值得保留
- **Long 反馈**:"写周报时候可以再进行一次重要性筛选和合并"
- **规则提炼**:daily 评级偏宽松,周报生成时再做一次收紧筛选,允许淘汰 daily 里被 ⭐⭐ 但不够分量的条目

### 2026-04-15(基于 4月14-15日实际标记)

**勾选的条目：**
- Waymo 伦敦公路测试启动（⭐⭐⭐ → [x]）
- Tesla Cybercab 量产爬坡 50+ 辆（⭐⭐ → [x]）
- Tesla FSD V14.3.1 发布（⭐ → [x]）← 小版本但有实质技术改进
- Tesla FSD 欧洲 HW3 集体维权（⭐ → [x]）← 负面/阻力事件
- Uber 追加投资 Lucid 扩大至 3.5 万辆采购承诺（⭐⭐ → [x]）

**未勾选的条目：**
- Lucid 新 CEO Silvio Napoli + $7.5 亿融资（⭐⭐ → [ ]）← 公司治理事务，非直接 robotaxi 推进

**规则提炼：**
1. **Tesla 小版本更新：看内容，不看版本号**：V14.3.1 有神经网络反应速度提升 20%、Autopilot 品牌重命名为 Self-Driving，属于实质性技术改进，值得勾选。纯 bug fix 或 UI 微调才是真正的"不勾"。规则修正：之前"小版本(v14.3.1)不算"是指不自动升为 ⭐⭐，但如果内容有实质意义，⭐ 也可以勾选。
2. **Tesla 商业化阻力事件值得关注**：HW3 集体维权直接影响 FSD 欧洲推广进程，属于"影响商业化进程的阻力/风险事件"，即使是负面新闻也值得勾选。
3. **OEM 公司治理事务（CEO 更换、融资）不如直接商业化行动重要**：Lucid 换 CEO + $7.5 亿融资虽然金额大，但属于公司内部治理，对 robotaxi 落地的直接推进有限，不如 Uber 扩大采购承诺（直接商业化行动）重要。判断标准：这件事是否直接推动 robotaxi 上路？

  - Tesla FSD 欧洲统计屏幕(功能改进,非突破性)
  - 小鹏 Mona M03 达 25 万辆里程碑(交付数字)
  - 小鹏 GX 技术发布会定档 4月15日(发布会预告)

**规则提炼**:
1. **实际商业落地测试 > 一切**:Uber+Nuro 旧金山测试是真实的商业化尝试,优先级最高
2. **监管突破(已批准)> 政策草案**:荷兰 FSD 批准值得勾选,但中国 L3 标准草案(未生效)不勾

5. **不勾选的类型**:

   - 发布会预告(4月15日发布会)
   - CEO 观点/趋势判断(L3 被跳过)
   - 产品外观设计(大蓝灯)
   - 政策草案(非最终批准)

**优先级排序**:实际落地 > 监管突破(已批准)> 重大功能发布 > 明确时间表的商业化计划 > 其他
